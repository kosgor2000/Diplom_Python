Komus IMS (учебный прототип) — Python + Tkinter + SQLite
========================================================

Что это
------
Демонстрационная информационная система управления запасами (IMS) по требованиям ТЗ:
- прогноз спроса (MA/ES/Croston для прерывистого спроса)
- расчёт параметров пополнения (min/max, safety stock, ROP) + ABC/XYZ
- ATP (Available-to-Promise) и умное резервирование с FEFO/FIFO
- партии/сроки годности (лотирование)
- workflow задач и уведомления
- интеграционный "входящий ящик" (эмуляция EDI/API) и обработка сообщений
- KPI-панель: OOS/OSA, оборачиваемость, неликвиды/обесценение

Запуск
------
1) Установите Python 3.10+
2) В папке проекта выполните:
   python main.py

База данных
-----------
При первом запуске автоматически создаётся файл:
  data/komus_ims.sqlite3
и выполняется автозаполнение тестовыми данными (SKU, поставщики, остатки, продажи).

Структура
---------
main.py          — GUI/запуск приложения
db.py            — инициализация схемы, подключение, сид-данные
services.py      — расчёты (прогноз, ABC/XYZ, ROP, safety stock, KPI, ATP/резервы)
ui_widgets.py    — небольшие общие виджеты/утилиты для Tkinter

Примечания
----------
- Проект предназначен для дипломной части (Глава 2) как работающий прототип.
- Это не промышленная система: интеграции EDI/API реализованы в виде эмуляции входящих сообщений.
